🚨 Common Installation Issues

⚠ Script stopped/exited after running

The script exits after installing each component. This is NORMAL and BY DESIGN, not a bug!

Solution Run the same curl command again. It's safe to re-run — it will pick up where it left off.

! "Password not showing when I type"

When Terminal asks for a password, nothing appears on screen — no dots, no asterisks, nothing. This is normal security behavior.

Solution Just type the password and press Enter. It's being entered even though you can't see it.

! "command not found: claude"

After installation completes, typing claude shows "command not found".

Solution Quit Terminal completely (Cmd + Q) and reopen it. The PATH needs to reload.

! SSO login — not seeing Spotify org

When visiting claude.ai, the user doesn't see the option to select Spotify as their organization.

Solution Disable adblockers (uBlock etc.) on claude.ai. Try incognito mode. Make sure they've joined the "okta-c4e-chat-and-code-users" group in Band Manager. Group sync can take 4-6 hours — try logging out of spotify.okta.com and back in.

! /login not working in Claude Code

The /login command in Claude Code doesn't show the right options or fails.

Solution Complete the claude.ai SSO login in browser first. Then close Terminal (Cmd+Q), reopen, and try /login again. Select "Claude account with subscription". Try 2-3 times — this is a known quirk.

! /status shows wrong account or "Not logged in"

After running /login, /status doesn't show "Organization: Spotify".

Solution Go back to claude.ai and make sure the Spotify org is selected (not a personal account). Run /login again and choose "Claude account with subscription".

! PATH not configured correctly

If claude still doesn't work after restarting Terminal.

Solution Run these two commands:
echo 'export PATH="$HOME/.local/bin:$PATH"' >> ~/.zshrc source ~/.zshrc

! Browser doesn't open for sign-in

Claude Code should open a browser window for authentication.

Solution If it doesn't open automatically, look for a URL in the Terminal output and copy/paste it into a browser. Sign in with @spotify.com email.

⚠ "Permission denied" errors

Mac security may block Terminal from accessing certain folders.

Solution Go to System Preferences → Privacy & Security → Full Disk Access → Add Terminal. May require restarting Terminal.

⚠ Process seems frozen/stuck

Installation appears to hang with no progress.

Solution Wait 30 seconds. If still stuck, press Ctrl + C to cancel, then run the install command again.

🔧 During the Workshop

? "How do I open Terminal?"

Solution Press Cmd + Space to open Spotlight, type "Terminal", press Enter.

? "What folder am I in?"

Solution Type pwd (print working directory) and press Enter.

? "How do I get to my Desktop?"

Solution Type cd ~/Desktop and press Enter.

? "How do I add my images?"

Solution Drag image files directly from Finder into the Terminal window. The file path will be pasted automatically.

? "Claude is asking for permission"

Solution Use arrow keys to select an option, then press Enter. For the workshop, "Yes" is usually the right choice.

? "How do I exit Claude Code?"

Solution Type /exit or press Ctrl + C twice. Or just close the Terminal window.

📦 What the Installer Does

The script runs once and installs everything:

  1. Claude Code — The main CLI tool
  2. Skills — Custom skills (/create-presentation-site, /frontend-design, /create-moodboard, /marketing-experience)
Important: If the script stops or fails, it's safe to run again — it will pick up where it left off. Full installation takes about 5 minutes.

🔐 SSO Login (After Install)

After the script finishes, users need to complete SSO login:

  1. Open claude.ai in browser (disable adblockers first!)
  2. Enter @spotify.com email → click "Continue with SSO"
  3. Select "Spotify" as the organization (NOT a personal account)
  4. In Terminal, type /login in Claude Code
  5. Select "Claude account with subscription"
  6. Verify with /status — should show "Organization: Spotify"
Note: The /login command may need to be run 2-3 times. This is a known quirk. If group access isn't working, the user may need to wait 4-6 hours after joining the Band Manager group, or log out of spotify.okta.com and back in.

🆘 Escape Hatches

! Something went very wrong

If Terminal is completely unresponsive or showing errors you can't fix:

Nuclear Option Close Terminal, reopen it, and start fresh. The install command is idempotent — running it again won't break anything.

⚠ Attendee can't get it working

If someone is truly stuck and holding up their experience:

Fallback Have them watch the demo and follow along visually. They can pair with a neighbor to see it in action. The install is saved — they can try again at home.
