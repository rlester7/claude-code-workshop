âš  Script stopped/exited after running

The script exits after installing each component. This is NORMAL and BY DESIGN, not a bug!

Solution Run the same curl command again. Run 1 = Homebrew, Run 2 = gcloud, Run 3 = Claude Code + Skills.

! "Password not showing when I type"

When Terminal asks for a password, nothing appears on screen â€” no dots, no asterisks, nothing. This is normal security behavior.

Solution Just type the password and press Enter. It's being entered even though you can't see it.

! "command not found: claude"

After installation completes, typing claude shows "command not found".

Solution Quit Terminal completely (Cmd + Q) and reopen it. The PATH needs to reload.

! gcloud auth - browser doesn't open

During gcloud authentication, a browser window should open automatically for sign-in.

Solution Look for a URL in Terminal starting with https://accounts.google.com â€” copy and paste it into your browser manually.

! gcloud auth - "Access Denied" or permission error

User gets an error during Google authentication.

Solution Make sure they've joined the "Claude Code Users" group in Band Manager first. Access can take a few hours to propagate.

! "gcloud not found" after first run

The first run only installed Homebrew. gcloud hasn't been installed yet.

Solution Run the curl command again. The second run installs gcloud.

! PATH not configured correctly

If claude still doesn't work after restarting Terminal.

Solution Run these two commands:
echo 'export PATH="$HOME/.local/bin:$PATH"' >> ~/.zshrc source ~/.zshrc

! Browser doesn't open for sign-in

Claude Code should open a browser window for authentication.

Solution If it doesn't open automatically, look for a URL in the Terminal output and copy/paste it into a browser. Sign in with @spotify.com email.

âš  "Permission denied" errors

Mac security may block Terminal from accessing certain folders.

Solution Go to System Preferences â†’ Privacy & Security â†’ Full Disk Access â†’ Add Terminal. May require restarting Terminal.

âš  Process seems frozen/stuck

Installation appears to hang with no progress.

Solution Wait 30 seconds. If still stuck, press Ctrl + C to cancel, then run the install command again.

ðŸ”§ During the Workshop

? "How do I open Terminal?"

Solution Press Cmd + Space to open Spotlight, type "Terminal", press Enter.

? "What folder am I in?"

Solution Type pwd (print working directory) and press Enter.

? "How do I get to my Desktop?"

Solution Type cd ~/Desktop and press Enter.

? "How do I add my images?"

Solution Drag image files directly from Finder into the Terminal window. The file path will be pasted automatically.

? "Claude is asking for permission"

Solution Use arrow keys to select an option, then press Enter. For the workshop, "Yes" is usually the right choice.

? "How do I exit Claude Code?"

Solution Type /exit or press Ctrl + C twice. Or just close the Terminal window.
