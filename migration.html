Already have Claude Code?

Here's how to switch to the new login.

Claude Code at Spotify is moving from Vertex AI to direct Anthropic access via SSO.

It's a 5-minute process. Simpler setup, faster access to new models.

Recommended

The easy way

Let Claude Code handle the migration for you.

1

Open Terminal and type claude

2

Copy the text below and paste it into Claude Code Claude will remove the old Vertex settings for you

3

Open claude.ai in your browser and log in with SSO Use your @spotify.com email, select Spotify org (disable adblockers first)

4

Back in Claude Code, type /login Select "Claude account with subscription"

Copy this and paste into Claude Code:
Please help me migrate my Claude Code setup from Vertex AI to SSO. Here's what needs to happen: 1. Remove all Vertex AI environment variables from my ~/.zshrc and ~/.zprofile: - CLAUDE_CODE_USE_VERTEX - ANTHROPIC_SMALL_FAST_MODEL - ANTHROPIC_DEFAULT_OPUS_MODEL - CLOUD_ML_REGION - VERTEX_REGION_CLAUDE_4_6_OPUS - ANTHROPIC_VERTEX_PROJECT_ID 2. Clean up my ~/.claude/settings.json: - Remove all Vertex-related entries from the "env" block (it should be empty: {}) - Remove any model override that references Vertex (like "claude-opus-4-5@20251101") 3. After making changes, show me what was changed so I can verify. Then I'll run /login to complete the SSO setup.
Manual Method

Prefer to do it yourself?

Here are the manual steps.

1

Join the new group in Band Manager You may have been auto-transferred from the old group

Join okta-c4e-chat-and-code-users
2

Open ~/.claude/settings.json and empty the env block

open -e ~/.claude/settings.json

Remove everything inside "env": { } so it looks like empty braces.

Delete any CLAUDE_CODE_USE_VERTEX, CLOUD_ML_REGION, ANTHROPIC_VERTEX_PROJECT_ID entries.

3

Close all Terminal windows and open a fresh one This ensures your environment is clean

Sign In

Log in with SSO

1

Open claude.ai in your browser Disable adblockers (uBlock etc.) first — they can block the login

2

Enter your @spotify.com email Click "Continue with SSO" or "Login with SSO"

3

Select Spotify as your organization Do NOT select a personal account if you have one

4

Open Terminal and type claude to start Claude Code Search for "Terminal" in Spotlight (Cmd + Space) if you can't find it

5

In Claude Code, type /login Select "Claude account with subscription"

If /login doesn't work the first time, try it 2-3 times. This is a known quirk.

Verify

Check that it worked

Type this in Claude Code:

/status

You should see:

Login method: Claude Enterprise Account

Organization: Spotify

Email: yourname@spotify.com

✓ If you see this, you're all set!

Troubleshooting

Stuck?

Common migration issues:

Not seeing Spotify org?

Disable adblockers on claude.ai. Try incognito mode. Make sure you joined the new Band Manager group.

Group access not working?

It can take 4-6 hours to sync. Try logging out of spotify.okta.com and back in.

/login not working?

Complete the claude.ai SSO login in browser first. Then close Terminal, reopen, and try /login again.

/status shows wrong account?

Make sure you selected the Spotify org (not personal). Run /login again and choose "Claude account with subscription".
